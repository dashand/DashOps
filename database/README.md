This directory contains the SQL scripts for initializing and managing the DashOps database.

### Critical Files

#### `init.sql`
The definitive initialization script for DashOps v1.0.0.
*   **Role**: Defines the complete database structure (Tables: Users, Tasks, History, Task Families) and default configuration (Admin user, default columns).
*   **Usage**: Automatically executed by Docker when building the database container for the first time.

#### `clean_backup.sql`
A "clean" restoration script.
*   **Role**: Used to reset the database to a specific known state.
*   **Feature**: It includes `DROP TABLE IF EXISTS` commands, meaning it will completely wipe the existing data before restoring the backup. Use with caution.

#### `full_backup.sql`
A raw full backup generated by `pg_dump`.
*   **Role**: Contains a snapshot of correct schema and data.
*   **Note**: Unlike `clean_backup.sql`, it may not contain instructions to drop existing tables. Restoring it on a database that already has tables might cause conflicts.

### Archive

#### `archive/`
This folder contains historical migration files (`migration_vX.sql`).
*   **Role**: Keeps a trace of schema evolution. These files have been consolidated into `init.sql` and are no longer needed for new installations.
